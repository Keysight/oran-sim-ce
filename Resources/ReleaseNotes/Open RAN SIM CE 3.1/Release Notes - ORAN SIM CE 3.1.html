About This Release

ORAN SIM CE 3.1 is an update built upon ORAN SIM CE 3.0, and it requires the prior installation of CE 3.0.

Key updates in CE 3.1 include the introduction of the CU isolation topology, enabling a wraparound configuration with either DUSim and CoreSIM or DUSim and DN. Additionally, enhancements have been made across several components:
  • Infrastructure: Added support for IPv6 and IPv4v6 management addresses.
  • User Plane now supports running IxStack DPDK in containers for Telco Cloud worker images.
  • LoadCore: Introduced Gx interface support for 4G standalone core, alongside various other feature enhancements.
  • DUSim: New control plane procedures added.
  • CUSim: Added support for HTTPS.
  • CoreSIM: Enhanced with MDT settings.

    • This update strengthens the platform’s infrastructure, user plane, and component functionality, improving overall flexibility and performance.

    Installation Notes

    ORAN SIM CE 3.1 provides a new middleware upgrade file. Please also note that upgrading a setup from pre-LC4.0 version to ORAN SIM CE 3.1 is not supported.
    ORAN SIM CE 3.1 also comes with new agent upgrades.
    AWS and Azure images are not distributed on the product download page. AWS image can be found on the AWS Marketplace, and both (AWS and Azure) can be shared by request by the engineering team. Please contact engineering at email LoadCore-Support to request access to these images.

    The REST API dashboards names have been changed. For automation users, in case your harness is checking the statistics, please use the REST API stats migration script to automatically convert the old naming into the new one.
    Please follow these steps to create a setup for running the product:

    If ORAN SIM CE 3.0 setup is not yet deployed, please use the below steps to deploy ORAN SIM CE 3.0 and configure an account

    1. Deploy the UI/middleware cluster OVA file
    2. Deploy the licensing OVA file (optional)
    3. After deployment you can access the UI using the MW machine IP and access it using a browser: https://XX.XX.XX.XX. If shortly after deploying the MW the UI is not accessible, you need to wait a couple of minutes for all the pods to start up.
    4. To access the UI, you will be prompted for a username and password. You will need to register a new user and then use it each time you log in.
    5. Connect to the ORAN SIM CE with your account. If a ORAN SIM CE 3.0 setup is not already available please use the above steps to deploy a new setup

    If ORAN SIM CE 3.0 setup is deployed, please use the below steps to upgrade (creating a backup of existing configs and raising memory to 24 GB RAM is recommended):

      Apply the patch using the UI:

    1. After connecting to the ORAN SIM CE 3.0 setup, find the "Gear menu" in the upper right corner and click on it to bring up the menu
    2. Then, click on "Software Updates"
    3. Click the "Browse" button and use the opened control to find the upgrade tar on the disk. If the ORAN SIM CE 3.0 environment is cloud (not on premise) please use the cloud tar.
    4. Click on the "Start Update" button to apply the patch and wait for the procedure to end.
    5. Refresh your ORAN SIM CE 3.0 page in the broswser
    6. Access the same menu and check the version of the setup again to make sure that the patch has been applied successfully.

      Apply the patch using the KCOS CLI:

    1. Connect to the ORAN SIM CE 3.0 setup with either SSH (on port 22: admin/admin user) or directly through the console (console -> admin/admin user). This will make available the KCOS CLI.
    2. Copy the update file to the MW using SCP. (admin/admin user). If the ORAN SIM CE 3.0 environment is cloud (not on premise) please use the cloud tar.
    3. Give the following command in the KCOS CLI pointing to the update tar that was copied on the MW machine: kcos deployment offline-install ./LoadCore-MDW-BUILD_VERSION.tar
    4. Wait for the process to finish. This might take several minutes.
    5. Refresh your LoadCore page in the broswser
    6. Access the UI and check the version of the setup to make sure that the patch has been applied successfully.

    If ORAN SIM CE 3.0 agents are not yet deployed, please use the below steps to deploy and configure

    1. Deploy the ORAN SIM CE 3.0 agent OVA file
    2. Add another NIC to the agent VM from the hypervisor configuration window. This will be the test NIC.
    3. Log in to the agent VM and configure the management interface, if needed
    4. Run the agent-setup.sh script and follow the steps in the interactive setup. This step is required to connect the agent to the middleware/UI cluster.
      5. #  sudo ./agent-setup.sh
    5. At this point, you should be able to see the newly created agent in the UI
    Repeat these steps to deploy as many agents as required.

    If ORAN SIM CE 3.0 agents are deployed on Virtual Environments, please use the below steps to upgrade

    1. After connecting to the ORAN SIM CE 3.0 setup, find the "Gear menu" in the upper right corner and click on it to bring up the menu
    2. Then, click on "Agent Management"
    3. Select the agents to be upgraded then click on "Update" button
    4. A new pop-up window is displayed. Click on the "Select update package for upload" button and find the upgrade tgz file on the disk
    5. Click on the "Start" button to apply the patch and wait for the procedure to end

    Known Limitations

    • Sometimes the LC MW UI hangs while it gets updated. Workaround: wait several minutes, refresh the UI and check the Keysight Load Core version.

    • On License Server the search bar from license statistics is not working. The workaround is to manually search the strings
