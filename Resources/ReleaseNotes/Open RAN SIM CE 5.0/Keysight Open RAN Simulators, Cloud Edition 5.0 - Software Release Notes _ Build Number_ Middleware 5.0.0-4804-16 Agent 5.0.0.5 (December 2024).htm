About This Release

ORAN Sim CE 5.0 is bringing a set of innovative features, expanded support, and enhancements designed to improve performance, efficiency, and user experience. An overview of the major updates from this release includes new Edge Computing support and MME running on multiple cores in FullCore topology, various features and enhancements for CoreSim, MCPTT support and impairment(network emulation) on userplane, agent update from UI and R17 support for DUSim and CUSim.


What's New in This Release

LoadCore

  • FullCore topology: Edge Computing - AF Traffic Influence support
  • FullCore and SBA Tester topologies: User-agent header support for SBA Nodes
  • FullCore and SBA Tester topologies: Added "Retrieve AM Policy Data" option to PCF node
  • FullCore topology: apiPrefix value can be configured for DUT NFs
  • FullCore topology: MME simulation is now using multiple cores if available
  • FullCore topology: 4G network initiated default bearer deletion (default bearer lifetime support)
  • FullCore topology: 4G network initiated Update Bearer procedure
  • FullCore topology: Support N4u interface on UPF in case SMF node is DUT
  • FullCore topology: Added PCO/ePCO definition
  • FullCore topology: Hide Agent assignment on the nodes which are grouped with other nodes
  • gNodeB simulation: Handling of Rx GTP-U Error Indication
  • RAN simulation: Capability to process 5GMM Status, 5GSM Status, EMM Status and ESM Status procedurese
  • UPF Isolation topology: Capability to configure user plane inactivity timer
  • Added support for PDF reports on NG-RAN from IPSec NG-RAN topology
  • Added support for PDF reports on IP endpoints

CoreSim

  • Configurable Index to RFSP IE value in Initial Context Setup Request support
  • Mobile Terminated SMS support
  • Capability to process the Reset messages initiated by the gNodeB/eNodeB which have the Reset Type set to UE-associated Logical NG-connection List for 5G usecase, respectively UE-associated logical S1-connection item for 4G case
  • GUTI Reallocation support
  • Capability to process 5GMM Status, 5GSM Status, EMM Status and ESM Status procedures
  • Generic UE Configuration Update support
  • Added support for PDF reports on AMF

DUSim

  • Support for ENDC Cell Activation Procedure
  • Support for ENDC configuration update initiated by eNB
  • Support for DU Initiated Error Indication (NSA & SA) procedure
  • Support for SCTP connection reestablishment
  • R17 Support

CUSim

  • R17 Support

User plane

  • MCPTT MC Voice and MC Video
  • Impairment/Network emulation
  • SRTP
  • Real payload support for Voice and Video OTT (Agent only)

Infrastructure

  • Support for AMD Epyc based servers
  • Allow IMS to run on top of IxStack in IP Endpoints tolopogy
  • Agent update from UI
  • Support for config rename
  • Added field for UE range name
  • Quick interface mapping for agent assignment

Installation Notes

ORAN SIM CE 5.0 provides a new middleware image. Please note that upgrading a setup from previous versions is not supported.
ORAN SIM CE 5.0 also comes with new agent images and upgrades.
AWS and Azure images are not distributed on the product download page. Both (AWS and Azure) can be shared by request by the engineering team. Please contact engineering at email LoadCore-Support to request access to these images.

The REST API dashboards names have been changed. For automation users, in case your harness is checking the statistics, please use the REST API stats migration script to automatically convert the old naming into the new one.

Please use the below steps to deploy ORAN SIM CE 5.0 OVA and configure an account

  1. Deploy the UI/middleware cluster OVA file
  2. Deploy the licensing OVA file (optional)
  3. After deployment you can access the UI using the MW machine IP and access it using a browser: https://XX.XX.XX.XX. If shortly after deploying the MW the UI is not accessible, you need to wait a couple of minutes for all the pods to start up.
  4. To access the UI, you will be prompted for a username and password. You will need to register a new user and then use it each time you log in.
  5. Connect to the ORAN SIM CE with your account.

If ORAN SIM CE agents are not yet deployed, please use the below steps to deploy and configure

  1. Deploy the ORAN SIM CE 5.0 agent OVA file
  2. Add another NIC to the agent VM from the hypervisor configuration window. This will be the test NIC.
  3. Log in to the agent VM and configure the management interface, if needed
  4. Run the agent-setup.sh script and follow the steps in the interactive setup. This step is required to connect the agent to the middleware/UI cluster.
    5. #  sudo ./agent-setup.sh
  5. At this point, you should be able to see the newly created agent in the UI
Repeat these steps to deploy as many agents as required.

If ORAN SIM CE 3.0 agents are deployed on Virtual Environments, please use the below steps to upgrade

  1. After connecting to the ORAN SIM CE 5.0 MW, find the "Gear menu" in the upper right corner and click on it to bring up the menu
  2. Then, click on "Agent Management"
  3. Select the agents to be upgraded then click on "Update" button
  4. A new pop-up window is displayed. Click on the "Select update package for upload" button and find the upgrade tgz file on the disk
  5. Click on the "Start" button to apply the patch and wait for the procedure to end

Known Limitations

  • When taking licenses from SLUM, instead of username, it appears an hexa code
